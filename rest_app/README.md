# Лабораторная работа №3: REST API для Глоссария на FastAPI

Проект представляет собой простой веб-сервис (REST API) для управления глоссарием технических терминов. API позволяет создавать, получать, обновлять и удалять термины.

Сервис разработан с использованием фреймворка **FastAPI** и использует **Pydantic** для валидации данных. В качестве хранилища данных выступает локальный JSON-файл.

## Основные возможности

* **Получение списка всех терминов.** (`GET /terms`)
* **Получение информации о конкретном термине.** (`GET /terms/{term}`)
* **Добавление нового термина.** (`POST /terms`)
* **Обновление существующего термина.** (`PUT /terms/{term}`)
* **Удаление термина.** (`DELETE /terms/{term}`)

---

## Технологический стек

* **Python 3.8+**
* **FastAPI** — веб-фреймворк для создания API.
* **Uvicorn** — ASGI-сервер для запуска приложения.
* **Pydantic** — для валидации данных и управления настройками.

## 1. Установка и настройка проекта

Чтобы развернуть проект локально, следуйте этим шагам.

Склонируйте репозиторий с GitHub на свой компьютер:
```bash

cd glossary-fastapi-lab
```

Bash

## 2. Запуск приложения
После установки всех зависимостей, запустите веб-сервер с помощью Uvicorn:

```Bash

uvicorn main:app --reload
```
- main: имя файла main.py.

- app: объект FastAPI, созданный внутри main.py.

- reload: флаг для автоматического перезапуска сервера при изменении кода.

##  3. Работа с API
FastAPI автоматически генерирует интерактивную документацию для API. Это лучший способ протестировать все возможности сервиса.

### 3.1. Интерактивная документация (Swagger UI)
Откройте в браузере страницу: http://127.0.0.1:8000/docs

Вы увидите интерфейс Swagger, где можно просмотреть все доступные эндпоинты, их параметры, а также отправлять запросы и получать ответы прямо из браузера.

### 3.2. Примеры использования API с помощью curl
Ниже приведены примеры, как можно взаимодействовать с API через командную строку с помощью утилиты curl.

#### 1. Добавление нового термина (POST)
Добавим термин "API":
```
Bash

curl -X 'POST' \
  '[http://127.0.0.1:8000/terms](http://127.0.0.1:8000/terms)' \
  -H 'accept: application/json' \
  -H 'Content-Type: application/json' \
  -d '{
  "term": "API",
  "definition": "Application Programming Interface. Интерфейс для взаимодействия программ друг с другом."
}'
```
Ответ сервера (201 CREATED):
```
JSON

{
  "term": "API",
  "definition": "Application Programming Interface. Интерфейс для взаимодействия программ друг с другом."
}
```
#### 2. Получение списка всех терминов (GET)
```
Bash

curl -X 'GET' '[http://127.0.0.1:8000/terms](http://127.0.0.1:8000/terms)' -H 'accept: application/json'
```
Ответ сервера (200 OK):
```
JSON

[
  {
    "term": "API",
    "definition": "Application Programming Interface. Интерфейс для взаимодействия программ друг с другом."
  }
]
```
#### 3. Получение конкретного термина (GET)
Получим информацию о термине "API":
```
Bash

curl -X 'GET' '[http://127.0.0.1:8000/terms/API](http://127.0.0.1:8000/terms/API)' -H 'accept: application/json'
```
Ответ сервера (200 OK):
```
JSON

{
  "term": "API",
  "definition": "Application Programming Interface. Интерфейс для взаимодействия программ друг с другом."
}
```
#### 4. Обновление термина (PUT)
Обновим определение для термина "API":
```
Bash

curl -X 'PUT' \
  '[http://127.0.0.1:8000/terms/API](http://127.0.0.1:8000/terms/API)' \
  -H 'accept: application/json' \
  -H 'Content-Type: application/json' \
  -d '{
  "definition": "Application Programming Interface. Это контракт, который одна программа предоставляет другой."
}'
```
Ответ сервера (200 OK):
```
JSON

{
  "term": "API",
  "definition": "Application Programming Interface. Это контракт, который одна программа предоставляет другой."
}
```
#### 5. Удаление термина (DELETE)
Удалим термин "API" из глоссария:
```
Bash

curl -X 'DELETE' '[http://127.0.0.1:8000/terms/API](http://127.0.0.1:8000/terms/API)' -H 'accept: application/json'
```
Ответ сервера:
Сервер вернет пустой ответ со статусом 204 No Content, что означает успешное удаление
